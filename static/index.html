How Auto Download Works

On launch, the app checks FFmpeg/ffprobe + yt-dlp (required) and Deno (optional). If anything is missing, it asks permission before downloading tools to your user runtime folder (no admin required).

Reel Captioning

Auto-captions need faster-whisper and torch. Speaker diarization also needs pyannote.audio plus a Hugging Face token. These Python packages are not auto-installed by the EXE runtime tools flow.

Troubleshooting Ideas

  1. If you chose Manual Install Only, click Auto Install Missing any time to opt in and retry.
  2. If auto-install fails, confirm internet access to GitHub/FFmpeg hosts, then click Auto Install Missing again.
  3. If only Deno fails, the app still works normally; install Deno manually for harder YouTube challenge/protection cases.
  4. If captioning is missing, install faster-whisper and torch in the same Python environment that runs the app.
  5. If speaker detection is missing, install pyannote.audio and provide a Hugging Face token in the Short Form Editor.
  6. If corporate/firewall rules block downloads, manually install FFmpeg, yt-dlp, and Deno using the links above.
  7. If security software quarantines binaries, allow the runtime folder and retry, then restart the app.
  8. For YouTube extraction issues, use Update yt-dlp (1-click) before retrying.
